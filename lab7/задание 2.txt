1. Потоки ввода/вывода

- Создадим файл, используя команду echo:
echo "text" > file.txt
- Используя команду cat, прочитаем содержимое всех файлов каталога etc, ошибки перенаправим в отдельный файл:
cat /etc/* > output.txt 2> errors.txt

2. Конвейрер (pipeline)

- Используем команду cut на вывод длинного списка каталога, чтобы отобразить только права доступа к файлам:
ls -l | cut -d ' ' -f 1
- Отправим в конвейере этот вывод на sort и uniq, чтобы отфильтровать все повторяющиеся строки:
ls -l | cut -d ' ' -f 1 | sort | uniq

3. Управление процессами

1 Изменение конфигурации службы SSH

- Откроем файл конфигурации:
sudo nano /etc/ssh/sshd_config
- Найдём строку PasswordAuthentication и установим:
PasswordAuthentication no
- Перезапустим службу:
- sudo systemctl restart ssh
Вернём параметр yes и выполним:
- sudo systemctl reload ssh

Различие между restart и reload: restart полностью перезапускает службу, тогда как reload просто перечитывает изменённый конфигурационный файл без остановки работы службы.

2 Создание файла с текстом через cat

- Используем команду:
cat > newfile.txt
- Введём текст и завершим команду комбинацией Ctrl+D.

4. Сигналы процессам

- Установим Midnight Commander:
sudo apt install mc
- Запустим Midnight Commander:
mc
- В другом терминале найдём PID процесса mc (число в строке где написано mc, но не grep... Число стоит после имени пользователя):
ps aux | grep mc
- Завершим процесс, передав сигнал 9. PID — идентификатор процесса mc, который мы нашли ранее:
sudo kill -9 PID
